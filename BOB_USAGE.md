def print_education_message(finding):
    """Print educational message for a finding"""
    check_id = finding.get('check_id', 'unknown')
    path = finding.get('path', 'unknown')
    line = finding.get('start', {}).get('line', 'unknown')
    message = finding.get('extra', {}).get('message', 'Secret detected')
    
    print(f"\n{RED}{BOLD}🚨 SECRET DETECTED{RESET}")
    print(f"{YELLOW}Location:{RESET} {path}:{line}")
    print(f"{YELLOW}Type:{RESET} {check_id}")
    print(f"{YELLOW}Why this matters:{RESET}")
    print(f"  {message}")
```

### 3. AWS Keys Detection Rule (`.guardkey/rules/aws-keys.yaml`)

**Bob Feature Used:** Code Mode + Compliance Mode

**Original Prompt:**
```
"Generate Semgrep rules for AWS_ACCESS_KEY detection. Include test cases 
for true positives and false positives. The rule must match all AWS key 
prefixes (AKIA, ASIA, AGPA, etc.) and be OWASP A02:2021 compliant."
```

**Bob's Contribution:**
- Generated the complete Semgrep YAML rule
- Researched all AWS access key prefixes (8 different patterns)
- Created the regex pattern that matches all variants
- Added OWASP A02:2021 metadata for compliance tracking
- Wrote the educational remediation message

**Time Saved:** ~4 hours
- Research on AWS key formats: 1.5 hours
- Regex pattern development and testing: 1.5 hours
- OWASP compliance verification: 0.5 hours
- Documentation and examples: 0.5 hours

**Complete Rule Generated by Bob:**
```yaml
rules:
  - id: guardkey-aws-access-key
    patterns:
      - pattern-regex: '(?:A3T[A-Z0-9]|AKIA|AGPA|AIDA|AROA|AIPA|ANPA|ANVA|ASIA)[A-Z0-9]{16}'
    message: AWS Access Key ID found in $FILE. This key grants programmatic access to your AWS account. Rotate this key immediately and use environment variables instead.
    languages: [generic]
    severity: ERROR
    metadata:
      owasp: A02:2021
      category: secrets
      remediation: |
        ❌ DON'T: hardcode keys in source code
        ✅ DO: Use environment variables
        Example: os.environ.get('AWS_ACCESS_KEY_ID')
```

**Bob's Compliance Review:**
After generating the rule, Bob performed a compliance check:

```
Prompt: "Review this AWS key detection rule against OWASP A02:2021 
Cryptographic Failures. Does it meet the standard?"

Bob's Analysis: "Yes, this rule aligns with OWASP A02:2021 by:
1. Detecting hardcoded credentials (A02 requirement)
2. Providing remediation guidance (security education)
3. Using ERROR severity for critical secrets
4. Including metadata for audit trails
The rule covers all 8 AWS key prefixes, reducing false negatives."
```
